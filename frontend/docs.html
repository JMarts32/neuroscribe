DOCUMENTATION

How NEUROSCRIBE transcribes, and what every option does. Defaults are tuned for this machine (AMD Ryzen 7 PRO 8840U, CPU).

How it works

Your file is decoded to 16 kHz mono audio, split into ~30 second windows, and fed to a Whisper speech-recognition model running through faster-whisper (the CTranslate2 engine). The model predicts text token by token; we collect the segments and print one [mm:ss] text line each. Everything runs locally on your CPU — nothing is uploaded.

Language

Auto-detect lets the model guess the spoken language from the first few seconds. Forcing a language is faster and avoids mis-detection on short or noisy clips. Whisper is multilingual; this app exposes a common subset.

Beam size

Beam search keeps the N most-likely transcription paths and picks the best. 1 is greedy decoding (fastest); 5 (default) explores more and is usually more accurate at a modest speed cost. Above ~5 gains are tiny.

VAD filter

Voice Activity Detection trims silent/non-speech stretches before transcription. Keeping it on (default) is faster and prevents the model from "hallucinating" text during long silences.

Recommended defaults for this PC

  • Model large-v3 — best accuracy; this CPU handles it.
  • Compute int8 — uses the 8840U's AVX512-VNNI, fastest on CPU.
  • Language English, Beam 5, VAD on.
  • Need a quick draft? Try distil-large-v3 or medium — much faster, still good.
